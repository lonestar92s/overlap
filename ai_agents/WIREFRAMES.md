## 19. User Flows

### Flow 1: Search â†’ Flight â†’ Create Trip â†’ Add Home Base

```
1. User searches for matches
2. Sees "Find flights" button
3. Clicks â†’ FlightSearchScreen
4. Searches flights
5. Selects flight â†’ Creates trip
6. Prompted to add home base
7. Adds home base
8. Sees travel times in match cards
```

### Flow 2: Existing Trip â†’ Add Home Base â†’ View Travel Times

```
1. User opens existing trip
2. Clicks "Manage Home Bases"
3. Adds home base with date range
4. System calculates travel times
5. Travel times appear in match cards
6. User can expand to see all options
```

### Flow 3: Multi-City Trip â†’ Multiple Home Bases

```
1. User has matches in London and Madrid
2. Adds "London (Jan 15-18)" home base
3. Adds "Madrid (Jan 19-22)" home base
4. System automatically uses correct home base for each match
5. Shows inter-match travel options between cities
```

### Flow 4: Inter-Match Travel Planning

```
1. User views trip with matches in different cities
2. Sees inter-match travel card
3. Clicks "View Travel Options"
4. Sees trains and flights
5. Selects option
6. Option saved to trip planning
```

## Implementation Notes

1. **Progressive Enhancement**: Features work without home bases (graceful degradation)
2. **Lazy Loading**: Travel times load on-demand, not all at once
3. **Caching**: Travel times cached to reduce API calls
4. **Offline Support**: Show cached travel times when offline
5. **Error Handling**: Clear error messages, retry options
6. **Accessibility**: Proper labels, touch targets, screen reader support
